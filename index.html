Application Web - EEG Explorer

Technologies utilisées : Python Dash Flask PostgreSQL CSS Bootstrap SQL Alchemy LangChain OpenAI

EEG Explorer est une application web en Python avec architecture multicouche, elle communique avec une base de données PostgreSQL via une API REST Flask. L’interface, conçue avec Dash, permet aux utilisateurs de poser des questions en langage naturel pour interroger une bases de données EEG. Grâce à Langchain et OpenAI, ces questions sont traduites en requêtes SQL, exécutées, puis reformulées en réponses compréhensibles. L’outil simplifie l’exploration de données EEG et valorise leur usage par l’IA.

Pipeline de traitement des données EEG

Pipeline de traitement des données EEG

Technologies utilisées : Python Matlab Pandas Numpy Plotly MNE Scikit-learn Tensorflow

Développement et mise en production d’un pipeline de traitement des données EEG en Python, permettant une analyse automatisée et reproductible de signaux neurophysiologiques issus de formats variés.

🔹 Analyse des fichiers EEG pour identifier le type de casque, le taux d’échantillonnage, le nombre et la position des électrodes selon le format d’entrée.
🔹 Prétraitement des données : filtres passe-bande, cut-off fréquentiel, normalisation.
🔹 Visualisation interactive des données : signaux continus, spectrogrammes, topomaps, pourcentages de PSD.
🔹 Calcul avancé d’indicateurs EEG :

  • ERCOH – cohérence de phase
  • PSD – densité spectrale de puissance
  • Fréquence du pic alpha
  • ERSP – perturbation spectrale liée à l'événement
🔹 Extraction de biomarqueurs EEG pathologiques basés sur la littérature scientifique.
🔹 Nettoyage automatisé des données EEG pour réduire le temps d’inspection manuelle et améliorer la qualité des analyses.

Dashboard Business Spectre

Dashboard business

Technologies utilisées : Looker SQL BigQuery MongoDB

Conception et mise en œuvre d’un dashboard business pour Spectre Biotech, destiné à améliorer le suivi de l’activité et la prise de décision stratégique.

  • Identification des besoins métiers et proposition de KPI pertinents.
  • Extraction des données anonymisées depuis MongoDB vers BigQuery pour analyses avancées.
  • Transformation des données via SQL dans BigQuery (jointures, agrégations, filtres).
  • Création de visualisations dynamiques et interactives dans Looker (dashboard multi-pages).

Contenu du dashboard :

  • Page Production : nombre et types d’utilisations, délais de rendu des rapports, patients (dont nouveaux).
  • Page Customer : comptes actifs/inactifs, nouveaux comptes.
  • Page Quality : taux de données complètes/incomplètes, électrodes rejetées.

🔍 Ce projet a permis de transformer des données brutes en indicateurs clairs et exploitables, facilitant le pilotage opérationnel de l’entreprise.
